# XHFrameworkSkill 技能编辑器：五类节点详解与火球术完整配置

本文基于 **XHFrameworkSkill** 项目实际代码，系统讲解技能编辑器中五类节点（Ability、Effect、Task、Condition、Cue）的作用、字段、连接关系与执行流程，并以「火球术」为完整示例串联全流程。

---

## 一、技能节点（Ability Node）

### 1.1 作用

技能节点对应 GAS 的 **GameplayAbility**，是整张技能图的**根节点**，负责：

- 定义技能身份（skillId、标签）
- 控制**能否激活**（消耗、冷却、标签条件）
- 控制**激活后行为**（授予标签、取消其他技能、运行中打断条件）
- 通过端口驱动：消耗、冷却、激活链、动画链、事件链

数据类：[`AbilityNodeData`](Assets/SkillEditor/Data/Ability/AbilityNodeData.cs)，运行时：[`GameplayAbilitySpec`](Assets/SkillEditor/Runtime/Ability/GameplayAbilitySpec.cs)。

### 1.2 必填与重要字段

| 字段 | 说明 | 必填建议 |
|------|------|----------|
| `skillId` | 技能 ID，用于查找与标识 | 建议填，运行时用 Graph 的 name 作 SkillId |
| `assetTags` | 技能自身标签 | 可选，用于识别技能 |
| `activationRequiredTags` | **激活所需标签**：拥有者必须拥有才能激活 | 按需 |
| `activationBlockedTags` | **激活阻止标签**：拥有则不能激活（如 CD、Skill.Running） | 火球术必配 CD 标签 |
| `activationOwnedTags` | 激活时授予的标签（如 Skill.Running） | 建议用于“技能进行中” |
| `cancelAbilitiesWithTags` | 激活时取消带有这些标签的技能 | 按需 |
| `blockAbilitiesWithTags` | 本技能运行时阻止哪些标签的技能激活 | 按需 |
| `ongoingBlockedTags` | **运行时阻止标签**：运行中若获得则取消技能（眩晕、沉默等） | 控制打断时必配 |
| `eventOutputPorts` | 事件输出端口列表（被动/事件触发） | 被动技能时配置 |

**火球术建议**：`activationBlockedTags` 中加入 `CD.火球术`、`Skill.Running`，避免连发与 CD 中释放。

### 1.3 激活策略

- **主动释放**：玩家输入 → `AbilityContainer.TryActivate(skillId)` → `GameplayAbilitySpec.CanActivate()` 检查（Required/Blocked 标签、消耗、冷却）→ `Activate(target)`。
- **被动/事件触发**：在 Ability 节点上添加「事件监听」端口（如受击时、造成伤害时、死亡时、自定义），端口名会写入 `AbilityEventPortData.PortId`；运行时 `Owner.OnTGameplayEvent` 触发时，`GameplayAbilitySpec.OnGameplayEvent` 根据 `eventType` 匹配端口并执行 `SpecExecutor.ExecuteConnectedNodes(SkillId, _abilityNodeGuid, PortId, _context)`。

事件类型见 [`GameplayEventType`](Assets/SkillEditor/Data/Base/NodeType.cs)（OnHit, OnDealDamage, OnTakeDamage, OnDeath, OnKill, Custom）。

### 1.4 与其他节点的连接关系

| 输出端口 | 连接对象 | 执行时机 |
|----------|----------|----------|
| **消耗** | CostEffect 节点 | 激活时立即执行，扣资源（如 MP） |
| **冷却** | CooldownEffect 节点 | 激活时立即执行，上 CD（普通/充能） |
| **激活** | 任意 Effect/Task/Condition/Cue | 激活后主逻辑链起点 |
| **动画** | Animation 节点 | 激活时播动画，并驱动时间轴上的「时间效果」与「时间 Cue」 |
| **事件端口**（如「受击时」） | Effect/Task/Cue | 对应游戏事件触发时执行 |

**注意**：消耗/冷却由 `GameplayAbilitySpec` 在 `Activate` 内按端口名 "消耗"、"冷却" 查找连线并执行；「激活」链由 `SpecExecutor.ExecuteConnectedNodes(..., "激活", _context)` 执行。

## 二、效果节点（Effect Node）

### 2.1 三种持续类型

在 [`EffectNodeData`](Assets/SkillEditor/Data/Base/EffectNodeData.cs) 与 [`NodeType`](Assets/SkillEditor/Data/Base/NodeType.cs) 中：

| 类型 | 枚举值 | 行为 |
|------|--------|------|
| **Instant** | `EffectDurationType.Instant` | 应用后立即执行「初始流程」+「完成流程」，不挂到 EffectContainer，不授予 grantedTags |
| **Duration** | `EffectDurationType.Duration` | 应用后加入目标 EffectContainer，在 `duration` 秒内有效，可周期执行；到期或移除时走「完成流程」 |
| **Infinite** | `EffectDurationType.Infinite` | 永久挂载，生命周期由逻辑显式移除（如投射物销毁、Buff 被驱散） |

冷却、Buff 一般为 **Duration**；伤害、治疗、消耗为 **Instant**；投射物效果在代码里被设为 **Infinite**，由投射物销毁时 `Remove()`。

### 2.2 如何修改属性（AttributeSet）

- 在 **Effect 节点** 的 `attributeModifiers`（或子类如 Cost/Damage/Heal 的专用字段）中配置：
  - `attrType`：目标属性（如 Health, Mana）
  - `operation`：Add / Multiply / Divide / Override
  - `magnitudeSourceType`：FixedValue / Formula / ModifierMagnitudeCalculation / SetByCaller
  - 公式可用 `Caster.Attack`、`StackCount` 等，见项目公式求值
- **Instant**：在 `ExecuteInitialFlow` 里直接改目标的 `Attribute.BaseValue`。
- **Duration/Infinite**：在 `Execute` 里把 `Modifier` 加到目标的 `Attribute` 上，并 `Recalculate`；移除时从 Attribute 上移除该 Effect 的 Modifier 再 Recalculate。

见 [`GameplayEffectSpec`](Assets/SkillEditor/Runtime/Effect/GameplayEffectSpec.cs) 的 `ExecuteInitialFlow`、`Execute`、`Remove`。

### 2.3 如何授予 Buff 与 GrantedAbility

- **授予标签**：Effect 的 `grantedTags` 在应用时加入目标 `OwnedTags`，移除时移除；用于配合「持续所需标签」「应用所需/免疫标签」等。
- **Buff**：使用 **BuffEffect** 节点（[`BuffEffectNodeData`](Assets/SkillEditor/Data/Effect/BuffEffectNodeData.cs)），默认 `Duration`、可堆叠、可周期；Buff 的「初始效果」「每周期执行」「完成效果」等端口连到其它 Effect/Cue。
- **GrantedAbility**：当前项目主要用「技能由 AbilityContainer 授予」；若需由 Buff 授予子技能，需在 Buff 的「初始效果」链上挂接授予逻辑或通过事件触发能力，具体取决于你是否扩展了 GrantedAbility 的配置（当前代码中未看到独立 GrantedAbility 节点，可视为预留扩展）。

### 2.4 效果节点通用输出端口

| 端口名 | 含义 | 执行时机 |
|--------|------|----------|
| 初始效果 | 应用时立即执行子节点 | Apply 时 |
| 每周期执行 | 周期触发（isPeriodic 且 period 到时） | 每个周期 |
| 刷新时 | 堆叠刷新时 | Refresh 时 |
| 完成效果 | 效果结束/移除时执行 | 到期或 Remove |
| 全部移除后 | 移除流程中执行 | Remove/Cancel 时 |
| 溢出 | 堆叠到上限且策略为 AllowOverflowEffect 时 | 堆叠溢出时 |

## 三、任务节点（Task Node）

### 3.1 常用任务类型

| 节点类型 | 数据类 | 作用 |
|----------|--------|------|
| **搜索目标** | `SearchTargetTaskNodeData` | 按形状（圆/扇/线）搜索目标，输出「对每个目标」「完成效果」 |
| **结束技能** | `EndAbilityTaskNodeData` | 结束当前技能（Normal/Cancel），无输出端口 |
| **动画** | `AnimationNodeData` | 被 Ability「动画」端口连接，提供动画名与时长，以及**时间效果**（按帧触发 Effect）和**时间 Cue**（按时间段播放 Cue） |

「播放动画」由 Ability 节点连到 **Animation 节点** 完成；**等待事件**通过 Ability 的**事件端口**（如受击时）触发子节点；**碰撞检测**由 **ProjectileEffect** 的「碰撞时」端口在 `ProjectileController` 命中时触发。

### 3.2 串联多个 Task

- 在图中用**连线**串联：Ability「激活」→ TaskA「输入」→ TaskA 的输出 → TaskB「输入」→ …
- 运行时：`SpecExecutor.ExecuteConnectedNodes(skillId, nodeGuid, outputPortName, context)` 会按连接顺序执行同一端口上的所有下游节点（同一端口可连多条线，执行顺序取决于 `GetConnectedNodes` 的返回顺序）。
- 例如：激活 → 搜索目标 → 「对每个目标」→ 伤害效果；最后再连到「结束技能」。

### 3.3 异步与回调

- **投射物**：ProjectileEffect 异步飞行，命中时回调 `OnProjectileHit`，内部执行「碰撞时」端口；到达目标位置执行「到达目标位置」端口；销毁时 `OnProjectileDestroy` 调用 `Remove()`。
- **动画时间轴**：Ability 每帧 `Tick` 根据动画播放时间驱动「时间效果」与「时间 Cue」端口，属于时间驱动的异步触发。
- Task 本身是**同步执行**的：Execute 一次执行完当前 Task 并立即执行其输出端口连接的节点；异步性由 Effect（如投射物、持续效果）或动画时间轴体现。

## 四、条件节点（Condition Node）

### 4.1 Tag 条件判断

- **技能层**：Ability 的 `activationRequiredTags` / `activationBlockedTags` 在 `CanActivate()` 中检查；`ongoingBlockedTags` 在运行中通过 `OwnedTags.OnTagAdded` 监听，命中则取消技能。
- **效果层**：Effect 的 `applicationRequiredTags` / `applicationImmunityTags` 在 `CanApplyTo(target)` 中检查；`ongoingRequiredTags` 在持续期间若目标失去所需标签会移除效果（见 `GameplayEffectSpec` 中 Tag 监听）。
- **表现层**：Cue 的 `requiredTags` / `immunityTags` 在 `CanPlayOnTarget(target)` 中检查。

### 4.2 Required Tags / Blocked Tags 区别

| 类型 | 含义 | 典型用途 |
|------|------|----------|
| **Required** | 必须拥有所列标签才能通过 | 仅对「己方」「可治疗」单位生效 |
| **Blocked / Immunity** | 拥有任一则不通过 | 免疫、不可选中、CD 中不释放 |

Ability 的「激活阻止」= 拥有则不能激活；Effect 的「应用阻止」= 目标拥有则不可应用；Cue 的「播放阻止」= 目标拥有则不播放。

### 4.3 运行时动态条件检查

- **属性比较条件**：[`AttributeCompareConditionNodeData`](Assets/SkillEditor/Data/Condition/AttributeCompareConditionNodeData.cs) + [`AttributeCompareConditionSpec`](Assets/SkillEditor/Runtime/Condition/AttributeCompareConditionSpec.cs)，比较目标属性与配置值（固定值或百分比），输出「是」/「否」两个端口，下游再接不同分支。
- 条件节点执行时 `Evaluate(target)` 得到 bool，再执行 `result ? "是" : "否"` 的连线节点，实现动态分支。

## 五、表现节点（Presentation / Cue Node）

### 5.1 类型与触发时机

| 类型 | 数据类 | 典型触发时机 |
|------|--------|--------------|
| **粒子特效** | `ParticleCueNodeData` | 连在 Effect「初始效果」/「完成效果」或动画时间轴「时间 Cue」 |
| **音效** | `SoundCueNodeData` | 同上 |
| **飘字** | `FloatingTextCueNodeData` | 伤害/治疗结果飘字，常连在 Damage/Heal 效果后，从 Context 取 `DamageResult` 等 |

触发时机完全由**连线**决定：连在「初始效果」即应用时播；连在「完成效果」即结束时播；连在动画时间轴的「时间 Cue」即在某段时间内播（可设开始/结束帧，`destroyWithNode` 时随节点结束停止）。

### 5.2 与逻辑层解耦

- Cue 只读上下文（Caster、Target、Context 数据），不修改属性、不挂 Effect；逻辑（伤害、治疗、Buff）全部由 Effect 节点完成。
- 表现通过 `positionSource`（Caster/MainTarget/ParentInput 等）和 `contextDataKey`（如 DamageResult）取位置与数值，逻辑层只需把结果写入 Context，Cue 只负责播放与生命周期。

### 5.3 多端同步

- 技能与效果的**执行权**在服务端或权威端，表现层只根据「技能激活」「效果应用」「命中」等事件做客户端表现；Cue 触发时机应与逻辑事件一致（例如命中时执行「碰撞时」→ 伤害 Effect → 飘字 Cue），这样多端只需同步事件或结果，各端本地触发相同 Cue 即可。
- 若 Cue 由服务端事件驱动，客户端收到事件后执行同一 SkillId+NodeGuid 的 Cue 节点，即可保证一致。

## 六、火球术完整配置流程（串联五类节点）

以下为在编辑器中从零做一发「火球术」的推荐配置流程（与现有 `火球术.asset` / `三火球.asset` 思路一致）。

### 6.1 新建技能图与 Ability 节点

1. 创建 SkillGraph 资源（或复制现有），命名为如 `火球术`（运行时 SkillId = 资源 name）。
2. 在图中创建 **技能** 节点（Ability），在 Inspector 中：
   - `skillId`：如 7001（或 0，用 name 标识即可）
   - `activationBlockedTags`：添加 `CD.火球术`、`Skill.Running`（需在标签资源中已有）
   - 其他标签按需（如 `activationOwnedTags` = Skill.Running）

### 6.2 消耗与冷却

1. 创建 **消耗效果** 节点（CostEffect），在属性修改器中添加一条：AttrType=Mana，Operation=Add，FixedValue=-10（或公式）。
2. 创建 **冷却效果** 节点（CooldownEffect），duration 设为 3（秒），grantedTags 添加 `CD.火球术`；若用充能，选 CooldownType.Charge 并设 maxCharges、chargeTime。
3. 连线：Ability **消耗** → CostEffect **输入**；Ability **冷却** → CooldownEffect **输入**。

### 6.3 动画与时间轴（可选）

1. 创建 **动画** 节点（Animation），选择 Spine 资源与动画名（如 Attack_01），设置动画帧数、是否循环。
2. 连线：Ability **动画** → Animation 节点 **输入**。
3. 若要在某一帧发球：在 Animation 的「时间效果」里加一条，triggerTime=发球帧，其端口连到 **投射物效果** 节点（见下）。

### 6.4 投射物与伤害（核心逻辑）

1. 创建 **投射物效果** 节点（ProjectileEffect）：
   - launchPositionSource = Caster，targetPositionSource = MainTarget（或 Position）
   - projectilePrefab、speed、collisionRadius、collisionTargetTags 等按需
2. 若从 Ability「激活」直接发球：Ability **激活** → ProjectileEffect **输入**。
   若从动画某帧发球：Animation 的对应**时间效果**端口 → ProjectileEffect **输入**。
3. 创建 **伤害效果** 节点（DamageEffect），配置伤害类型、数值（固定或公式/MMC），targetType = ParentInput（用投射物传入的命中目标）。
4. 连线：ProjectileEffect **碰撞时** → DamageEffect **输入**（命中时对当前目标造成伤害）。

### 6.5 表现（Cue）

1. 创建 **粒子特效** Cue：选火球飞行/命中预制体，positionSource = ParentInput（或 Caster/MainTarget），连到 ProjectileEffect「初始效果」或时间轴「时间 Cue」做飞行尾迹；连到「碰撞时」后做命中爆炸。
2. 创建 **音效** Cue：连在「初始效果」或「碰撞时」。
3. 创建 **飘字** Cue：FloatingTextType=Damage，contextDataKey=DamageResult，连在 DamageEffect 之后（或「完成效果」），这样伤害结果会从 Context 传入飘字。

### 6.6 结束技能

1. 创建 **结束技能** 任务节点（EndAbilityTask），endType = Normal。
2. 连线：DamageEffect **完成效果** → EndAbilityTask **输入**（伤害结算后结束技能）。若没有「完成效果」链，也可从 ProjectileEffect「到达目标位置」或「完成效果」连到 EndAbilityTask，视你希望「何时」结束技能而定。

### 6.7 条件节点（可选）

若希望「仅当目标生命低于 30% 时造成额外效果」：
1. 创建 **属性比较** 条件节点：compareAttrType=Health，compareOperator=Less，compareValueType=Percentage，compareValue=30，percentageBaseAttrType=MaxHealth。
2. 将条件节点接在「碰撞时」与伤害之间：ProjectileEffect **碰撞时** → AttributeCompareCondition **输入**；Condition **是** → DamageEffect（或高伤版）；Condition **否** → 普通 DamageEffect 或直接结束。

### 6.8 流程小结（火球术）

```text
Ability [技能]
  ├─ 消耗 → CostEffect [扣蓝]
  ├─ 冷却 → CooldownEffect [上 CD]
  ├─ 动画 → Animation [播施法动画，可选]
  │         └─ 时间效果(某帧) → ProjectileEffect [生成火球]
  ├─ 激活 → ProjectileEffect [或直接从这里发球]
  │         ├─ 初始效果 → ParticleCue / SoundCue [飞行表现]
  │         ├─ 碰撞时 → DamageEffect [命中伤害]
  │         │            ├─ 完成效果 → FloatingTextCue [飘字]
  │         │            └─ 完成效果 → EndAbilityTask [结束技能]
  │         └─ 到达目标位置 → (可选) EndAbilityTask
  └─ activationBlockedTags: CD.火球术, Skill.Running
```

按上述顺序在编辑器中创建节点并连线，即可得到一套完整的火球术配置；再结合标签与属性配置即可扩展到更多技能与 Buff。
